# 代码评审技能

## 使用场景
- 提交最终代码变更之前
- 完成一组计划步骤之后
- 合并或上线代码之前

## 严重等级
- **Blocker（阻断级）**：行为错误、安全问题、存在数据丢失风险、测试/构建失败
- **Major（严重级）**：潜在 Bug、缺失边界情况处理、可靠性差
- **Minor（一般级）**：代码风格、清晰度、轻微可维护性问题
- **Nit（建议级）**：可选的优化与润色

## 检查清单
1. 功能正确性与需求匹配
2. 边界情况与异常处理
3. 测试（测试覆盖率充足、断言有意义）
4. 安全（密钥、权限认证、注入风险、不安全默认配置）
5. 性能（明显性能热点、N+1 查询、无效冗余操作）
6. 可读性与可维护性
7. 按需更新文档 / 注释

## 输出格式
- 阻断级问题
- 严重级问题
- 一般级问题
- 建议级问题
- 整体总结 + 后续操作
