# 技能名称：TDD 技能
## 名称
superpowers-tdd
## 描述
遵循测试先行原则（红/绿/重构），并为 Bug 添加回归测试。在实现功能、修复 Bug 或重构代码时使用。

---

## 适用场景
- 可进行单元测试的新功能
- 修复 Bug（只要可行，始终添加回归测试）
- 代码重构（先用测试保护原有行为）

## 规则
- 优先遵循 **红 -> 绿 -> 重构** 流程。
- 如果编写测试困难，仍需添加**验证方式**：最简复现脚本、集成测试或清晰的手动验证步骤。
- 保持测试聚焦：尽可能一个测试只验证一种行为。
- 测试命名以**行为**为准，而非实现细节。

## 流程
1. 定义行为变更（完成后应满足什么效果）。
2. 编写/修改测试用例来覆盖该行为（尽可能先让测试失败）。
3. 编写最简代码使测试通过。
4. 如需则进行重构（保持测试持续通过）。
5. 运行相关测试套件 + 所有代码检查工具。

## 输出要求
修改代码时需包含：
- 新增/修改了哪些测试
- 如何运行这些测试
- 这些测试验证了什么
