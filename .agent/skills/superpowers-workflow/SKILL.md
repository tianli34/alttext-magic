# 超级能力工作流

本技能定义了软件开发与自动化任务的**默认标准流程**。

## 何时使用本技能
当用户要求执行以下操作时使用：
- 编写或修改代码
- 调试错误或失败的测试
- 重构或提升代码质量
- 设计自动化工作流（如 API 集成、ETL、流水线）
- 添加测试、可靠性或安全校验

## 激活标记（必须执行）
读取本技能后（在任何其他输出之前），立即运行：

python .agent/skills/superpowers-workflow/scripts/record_activation.py --skill superpowers-workflow

## 默认流程（强制使用，除非明确无需）
1. **头脑风暴（简短）**：明确目标、约束、风险与验收标准。
2. **编写计划**：拆分为小步骤（每步 2–10 分钟），包含文件与验证方式。
3. **实现**：做最小且正确的修改；可行时优先**测试先行**。
4. **评审检查**：正确性、边界情况、安全、代码风格、可维护性。
5. **完成**：执行验证命令，汇总修改内容与后续步骤。

## 决策树：需要多少流程？
- **微小修改（1 个文件，逻辑明显）**：
  - 简短头脑风暴（3 条要点）→ 简短计划（3–5 步）→ 实现 + 验证。
- **非微小修改**：
  - 完整头脑风暴 + 完整计划后再编码。
- **高风险修改**（鉴权、支付、生产数据、安全、数据迁移）：
  - 增加明确风险控制：回滚方案、预演、额外测试、日志、安全默认值。

## 输出规则（沟通方式）
- 存在歧义时，必须明确说明**假设**。
- 必须包含**验证方式**（命令、测试或可观测检查）。
- 如需提问，**最多提 3 个**，然后按最优假设继续。

## 停止条件
出现以下情况时暂停实现，回到规划阶段：
- 需求存在冲突
- 关键未知项影响正确性
- 若无保护措施，修改可能导致数据丢失或安全问题
