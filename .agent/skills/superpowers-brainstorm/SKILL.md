# 头脑风暴技能

## 使用场景
- 在开发重要功能之前
- 在范围不明确的重构之前
- 在调试复杂问题之前
- 在设计自动化工作流之前

## 头脑风暴模板（严格使用此结构）
### 目标
- （1–2 句话）

### 约束条件
- （技术栈、时间、兼容性、性能、禁止改动的内容等）

### 已知背景
- （现有系统情况；相关文件/组件；当前行为）

### 风险
- （安全、数据丢失、功能退化、意外副作用）

### 可选方案（2–4 个）
每个方案包含：
- 概述
- 优点 / 缺点
- 复杂度 / 风险

### 推荐方案
- 选择一个方案并说明理由

### 验收标准
- 可验证结果的列表
