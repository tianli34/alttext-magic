# 调试技能

## 使用场景
- 运行时错误、不稳定测试、结果错误
- “之前正常，现在出问题”的功能退化
- 性能或超时问题（初步排查）

## 调试流程（不可跳过任何步骤）
1. **复现**
   - 捕获精确的错误信息、输入、环境、执行命令。
2. **最小化**
   - 精简到最小可复现案例（单个文件、单个函数、最小数据集）。
3. **提出假设（2–5 个）**
   - 按可能性高低排序。
4. **埋点观测**
   - 添加临时日志/断言，或使用现有诊断工具。
5. **修复**
   - 用最小改动消除根本原因。
6. **预防**
   - 添加回归测试或永久防护/校验逻辑。
7. **验证**
   - 运行原失败用例 + 相关测试套件。

## 报告格式
- 现象
- 复现步骤
- 根本原因
- 修复方案
- 回归防护措施
- 验证结果
