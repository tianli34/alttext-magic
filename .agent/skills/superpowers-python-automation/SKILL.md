# Python 自动化技能

本技能提供具体的 Python 实现范式，用于构建健壮的 REST API 自动化流程。

## 适用场景
- 调用一个或多个 REST API 的 Python 脚本
- ETL 任务、数据同步工具、Webhook 处理器
- 集成外部系统的命令行工具或小型服务

## 推荐技术栈（默认选型）
- HTTP 客户端：**httpx**（首选）或 requests
- 配置管理：环境变量 + 可选的 `.env` 文件，适当时结合 pydantic-settings 使用
- 日志记录：标准库 `logging`，搭配类结构化字段
- 测试框架：pytest（适用时，结合 respx 模拟 httpx 请求）

若项目已使用其他工具，遵循项目既有规范。

---

## 参考架构（轻量且可扩展）
- `client.py`：API 客户端封装（认证头、重试逻辑、分页辅助函数）
- `models.py`：带类型注解的载荷模型（dataclasses 或 pydantic 实现）
- `sync.py`：编排逻辑（数据获取 → 转换 → 新增/更新）
- `main.py`：命令行入口
- `tests/`：转换逻辑 + 客户端行为的单元测试

## HTTP 规则（强制遵循）
- 始终设置超时（连接超时 + 读取超时）
- 将请求发送逻辑集中到单个函数，确保重试/日志行为一致
- 绝不记录敏感信息（Authorization 头、令牌等）

### 重试策略指引
以下场景需重试：
- 网络错误/超时
- 429 状态码（存在 Retry-After 响应头时需遵守）
- 500–599 状态码
可选重试：408 状态码；仅当操作安全且语义明确时，可重试 409 状态码

以下场景**禁止**重试：
- 大部分 400–499 状态码（除非明确确认安全）

### 超时设置
- 显式设置超时时间，不依赖默认值
- 连接超时设为较小值，读取超时设为适中值

## 分页处理范式
至少实现一个分页辅助函数，支持：
- 响应中的 `next` URL 分页
- 响应中的游标令牌（cursor token）分页
- 页码/条数（page/limit）参数分页

添加强制终止机制：
- 最大页数 OR 最大处理条数 OR 最长耗时

---

## 幂等性实现范式（Python 版）
选择并文档化以下方案：
- 服务商支持时，使用 `Idempotency-Key` 请求头
- 基于稳定的 `external_id` 实现新增/更新（upsert）
- 持久化轻量级状态存储：
  - 最简方案：SQLite 文件（开源项目推荐）
  - 替代方案：JSONL 日志 + 日志压缩

最低要求：确保重复运行不会产生重复数据。

---

## 可观测性（Python 版）
日志最低要求包含：
- `run_id`（运行唯一标识）
- 请求信息：方法、URL/路径、状态码、耗时（毫秒）、重试次数
- 记录数统计：已处理/新增/更新/跳过/失败

同时需输出最终的汇总日志行。

---

## 验证要求
非简单功能需满足：
- 为数据映射/转换逻辑编写单元测试
- 至少编写一个分页或重试行为的测试（模拟场景）
- 增加 “试运行（dry-run）” 命令行标识（仅打印预期写入操作，不实际执行）
