# REST 自动化技能

本技能用于在构建调用 REST API 的自动化任务时，保证可靠性与安全性。

## 使用场景
当任务涉及以下内容时使用：
- 调用外部 REST API（增删改查、搜索、同步）
- 集成两个及以上系统（ETL、类 iPaaS 流程）
- Webhook、轮询或定时任务
- 数据采集、规范化、 enrichment、去重

## 默认设计原则
- **设计幂等**：重复执行不应产生重复数据或损坏数据
- **可观测**：日志/指标可关联每一次运行与每一次 API 调用
- **安全失败**：处理部分失败；避免静默数据丢失
- **感知限流**：合理退避并遵守服务商限制
- **最小权限**：安全处理密钥，避免权限范围过大

## 检查清单（若无特殊情况均需应用）

### 1）定义契约
- 输入（格式、必填字段、校验规则）
- 输出（数据去向、预期结构）
- 成功标准（“完成”的定义）
- 非目标（自动化不做的事情）

### 2）认证与密钥
- 明确认证类型：API Key、OAuth2、JWT、mTLS
- 严禁在代码或日志中硬编码密钥
- 支持通过环境变量 / 密钥管理工具注入密钥
- 如适用，规划令牌刷新（如 OAuth2）

### 3）幂等性与去重
至少选择一种：
- 使用服务商提供的幂等键（如支持）
- 使用稳定外部 ID（如 `external_id` 字段）做更新插入
- 维护本地/状态存储，映射源 ID → 目标 ID
- 无稳定 ID 时，使用确定性哈希去重
**必须明确文档化幂等策略**。

### 4）分页与增量同步
- 识别分页方式：`next` 链接、游标、页码+条数、偏移量+条数
- 确保循环安全终止（最大页数 / 最大时长）
- 尽可能使用 `updated_since`/ETag/If-Modified-Since 做增量同步
- 处理乱序更新与延迟到达的事件

### 5）重试、退避与超时
- 设置连接/读取**超时**
- 对瞬时错误重试：网络故障、429、5xx（带次数限制）
- 尽可能使用带抖动的指数退避
- 多数 4xx 错误**不重试**（根据语义，408/409/429 除外）
- 限制重试次数，并清晰暴露失败信息

### 6）限流与配额
- 遵守 `Retry-After` 与限流响应头
- 遇到 429 时实现自适应退避
- 考虑使用批量接口减少调用量
- 避免突发并发，除非明确安全

### 7）数据映射与校验
- 显式映射层（源数据 → 规范化 → 目标数据）
- 校验必填字段与类型
- 规范化通用格式（日期、枚举、货币、地区）
- 处理空值与部分载荷
- 记录被拒绝的数据及原因（不静默丢弃）

### 8）错误处理策略
为每类错误选择并文档化处理方式：
- **记录日志并跳过**（非关键单条数据）
- **重试**（瞬时错误）
- **隔离**（存储失败载荷后续处理）
- **终止运行**（系统性问题）
确保工作流最后输出清晰的执行总结。

### 9）可观测性与审计日志
最低要求：
- 运行 ID / 关联 ID
- 每次请求日志：请求方法、路径（不含完整密钥）、状态码、耗时、重试次数
- 计数器：已处理、新增、更新、跳过、失败条数
优先使用结构化日志（JSON）。

### 10）Webhook（如涉及）
- 验证签名（如提供）
- 防重放（事件 ID 幂等）
- 快速响应；必要时异步处理
- 存储原始事件载荷（可选但推荐）

### 11）安全控制
- 试运行模式（不写入数据）
- 范围限制（每次运行最大处理条数）
- “紧急停止”配置开关
- 破坏性操作的备份/回滚方案

## 输出要求（交付方案时必须包含）
- 幂等策略（1–3 条要点）
- 重试/退避策略
- 分页/增量同步方案（如适用）
- 错误处理策略 + 日志/隔离规则
- 验证方案（测试或安全沙箱运行计划）
