# 超级技能规则（始终生效）
除非用户明确选择退出，否则这些规则适用于**所有工作**。

## 1）非小型修改必须经过规划关卡
如果任务不只是极小改动，**禁止立即编辑代码**。
你必须：
1. 进行简要头脑风暴（目标、约束、风险、验收标准）
2. 编写包含验证步骤的分步计划
3. 请用户批准该计划

**只有获得批准后，方可开始实现**。

### 执行计划关卡（保持超级技能一致）
用户批准计划后，**禁止自动开始实现**。
你必须暂停，并提示用户执行：
`/superpowers-execute-plan`

**只有在调用该命令后**，才开始实现；
除非用户明确表示无需此命令直接继续。

### 什么算作“小型修改”？
- 仅修改单个文件
- 改动非常明确
- 风险极低

即便如此，仍需编写**简易计划（3–5 步）**，并包含验证环节。

## 2）验证为强制要求
实现完成后，你**必须**提供：
- 用于验证的精确命令（测试、代码检查、运行等）
- 若已执行，附上对应结果

## 3）优先使用 TDD / 回归测试
- 修复 Bug 时：如可行，添加回归测试
- 新增功能时：如可行，添加/调整测试

如果无法编写测试，需提供**具体可替代的验证方式**。

## 4）必须经过代码审查
在最终回复前，执行审查，并按严重程度列出问题：
- 阻塞级 / 主要 / 次要 / 建议优化

## 5）安全要求
- 严禁打印、记录密钥等敏感信息
- API 自动化需添加超时、重试和幂等性保障
- 安全失败机制（禁止无声的数据丢失）

## 产物持久化（强制）
所有头脑风暴、计划、审查、收尾输出**必须写入文件**，路径为：
`artifacts/superpowers/`

不要只保留在 IDE 内。
写入后，确认文件已存在。

## 持久化强制执行
当工作流需要将产物保存到 `artifacts/superpowers/` 时，
你**必须确保文件已写入磁盘**。

推荐方式：
`python .agent/skills/superpowers-workflow/scripts/write_artifact.py --path <...>`

若无法执行命令，指导用户**手动保存输出内容**。
