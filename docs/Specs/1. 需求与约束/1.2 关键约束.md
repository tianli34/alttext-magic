
- 写回成功率目标 >99%
- Shopify API 限流下可稳定运行：Bulk + 队列 + 重试 + 断点续跑
- `fileUpdate` 全局生效，同一媒体资产被多处引用时必须能提示影响范围
- 图片不落盘，仅保存 URL、ID、草稿文本、必要上下文与审计
- 生成前实时真值复核必须成立：
  - 已被外部补齐 Alt 的图片不得再调用 AI
  - 不得发生“已非空仍扣费”
- 共享文件在多使用场景下必须使用中性上下文，避免误写
- 展示模型必须区分：
  - **唯一写回对象**
  - **group 级展示投影**
- 产品媒体必须有稳定的 `position_index` 持久化口径，用于 UI 展示“图片位置”
- 额度一致性：
  - `credit_bucket / credit_reservation / credit_ledger / billing_ledger` 可交叉验证
  - 无可用额度时绝不调用 AI
- Free 配额必须能由账本自证：
  - 周期定义明确
  - 发放不可重复
  - 回归 Free 时行为可追溯
- 年付额度必须能由账本自证：年度发放、年度消费、年度到期
- Bulk 解析必须显式处理扁平化输出和 `__parentId`
- Bulk 结果 URL 过期必须可恢复，不得整 job 直接失败
- Bulk 结果解析必须流式下载、流式解析，避免大文件内存溢出
- 再次访问必须稳定读取**最近一次已发布结果**
- `PARTIAL_SUCCESS` 不能误删失败资源类型的历史有效数据
- `scope_flags` 改动后，前台与自动增量必须立即按新 scope 生效
- 增量刷新必须尽量避免无效重扫
- `image_usage` 必须支持 sweep，避免历史脏数据污染分组统计
- 装饰性标记必须在扫描、统计、生成、写回全链路生效
- 周期性额度桶必须具备**防重复发放约束**
