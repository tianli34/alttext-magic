`GET /api/bootstrap`

- 返回：
  - 当前计划与额度摘要
  - `includedRemaining`
  - `includedPeriodType`
  - 是否需要说明确认
  - 当前 `notice_version`
  - `lastPublishedAt`
  - 是否存在最近扫描结果
  - 是否有 RUNNING 的 scan job
  - `scanScopeFlags`
  - `lastPublishedScopeFlags`
  - `effectiveReadScopeFlags`

`POST /api/settings/scope`

- body:
  - `scopeFlags`
- 行为：
  - 仅更新 `shops.scan_scope_flags`
  - 不触发扫描
  - 不修改 `last_published_scope_flags`
- 返回：
  - `scanScopeFlags`
  - `lastPublishedScopeFlags`
  - `effectiveReadScopeFlags`

`POST /api/scan/start`

- body:
  - `scopeFlags`
  - `noticeVersion`
- 行为：
  - 若当前版本未确认，则先写 `scan_notice_ack`
  - 若 body 带 `scopeFlags`，则先更新 `shops.scan_scope_flags`
  - 获取 scan 锁
  - 创建 `scan_job`
- 说明：
  - `scopeFlags` 不是唯一修改入口；独立设置请使用 `/api/settings/scope`

`GET /api/scan/status?scanJobId=...`

- 返回 task 进度
- 返回当前 attempt 状态与错误
- 返回 publish 状态
- 返回前台当前仍应读取的 `lastPublishedAt`

`GET /api/dashboard`

- 返回当前 `effective_read_scope_flags` 内的分组统计：
  - `total / hasAlt / missing / decorative`
- 返回 `lastPublishedAt`
- 返回当前是否有重新扫描进行中
- 返回计划与额度摘要
- 返回 scope 元信息

`GET /api/candidates?group=PRODUCT_MEDIA&status=MISSING`

- 支持 group：
  - PRODUCT_MEDIA
  - FILES
  - COLLECTION
  - ARTICLE
- 支持 status：
  - `MISSING`
  - `GENERATION_FAILED_RETRYABLE`
  - `GENERATED`
  - `WRITEBACK_FAILED_RETRYABLE`
- 仅返回 `effective_read_scope_flags` 内数据
- 查询基于 `candidate_group_projection`
- 每项返回：
  - `usageCountPresent`
  - `contextMode`
  - `primaryUsage`
    - `type`
    - `id`
    - `title`
    - `handle`
    - `positionIndex`
  - `additionalUsageCount`
  - `impactScopeSummary`

`GET /api/candidates/:altCandidateId/usages?group=PRODUCT_MEDIA`

- 返回该 candidate 当前 `PRESENT` 的 usage 列表
- 支持按当前 group 过滤相关 usage
- 每项返回：
  - `usageType`
  - `usageId`
  - `title`
  - `handle`
  - `positionIndex`
- 用于前台展开“影响范围”详情

`POST /api/decorative/mark`

- `{ altTargetId }`

`POST /api/decorative/unmark`

- `{ altTargetId }`

`POST /api/generation/preflight`

- `{ altCandidateIds: [] }`
- 返回：
  - 预计消耗上限
  - `includedRemaining`
  - `includedPeriodType`
  - 欢迎额度剩余
  - 超额包额度剩余
  - 是否足够

`POST /api/generation/start`

- `{ altCandidateIds: [] }`
- 前置：
  - 通过 preflight
  - candidate 状态在 `MISSING / GENERATION_FAILED_RETRYABLE`
  - candidate 在 `effective_read_scope_flags` 内
  - 创建 reservation
- 返回 `generation_batch_id`

`POST /api/draft/update`

- `{ altCandidateId, editedText }`

`POST /api/writeback/start`

- `{ altCandidateIds: [] }`
- 前置：
  - candidate 状态在 `GENERATED / WRITEBACK_FAILED_RETRYABLE`
  - candidate 在 `effective_read_scope_flags` 内
- 返回 `writeback_batch_id`

`GET /api/history`

- 返回审计记录

`GET /api/billing/summary`

- 返回当前计划
- `includedRemaining`
- `includedPeriodType`
- 欢迎额度余额
- 超额包余额
- 最近购买记录
- 可选返回 `includedBuckets[]` 供账单页解释

`POST /api/billing/change-plan`

- `{ planCode, billingInterval }`

`POST /api/billing/purchase-pack`

- `{ planCode }`
- 仅付费计划可用

`GET /api/sse?batchId=...`

- 返回 `{ current, total, status }`
