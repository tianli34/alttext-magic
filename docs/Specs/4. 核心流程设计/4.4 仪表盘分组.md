
前台仪表盘只读取**已发布结果**，且只显示 `effective_read_scope_flags` 内的资源类型。

MVP 推荐统一基于 `candidate_group_projection` 统计，以保证：

- Dashboard 分组计数
- 候选列表
- 共享文件影响范围提示

三者口径一致。

分组维度：

- `group_type = PRODUCT_MEDIA`
- `group_type = FILES`
- `group_type = COLLECTION`
- `group_type = ARTICLE`

每组统计：

- `total`：该 group 下 projection 数
- `hasAlt`：对应 target `current_alt_empty = false`
- `missing`：对应 target `current_alt_empty = true` 且无激活 decorative mark
- `decorative`：对应 target `current_alt_empty = true` 且有激活 decorative mark

前台还应展示：

- `lastPublishedAt`
- 当前是否存在 RUNNING 的 `scan_job`
- 当前 `scan_scope_flags`
- 当前 `effective_read_scope_flags`
- 若有进行中的重新扫描，展示“正在重新扫描，当前显示的是上一次已完成结果”

> 同一共享文件若同时出现在 Product Media 与 Files 视角，可同时计入两组；但 candidate 仍只有 1 条，生成与写回只发生一次。  
> out-of-scope 分组不展示，也不允许从前台进入操作流。
