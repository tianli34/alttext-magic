
#### 4.8.1 触发

`POST /api/writeback/start { altCandidateIds }`

#### 4.8.2 前置校验

- 当前 shop 无 `SCAN / GENERATE` 锁
- candidate 资源类型必须在 `effective_read_scope_flags` 内
- candidate 状态必须为：
  - `GENERATED`
  - 或 `WRITEBACK_FAILED_RETRYABLE`
- 存在 draft
- 非 decorative

#### 4.8.3 写回流程

**Step A：二次读校验**

读取线上当前值：

- `FILE_ALT`：读取 `MediaImage.alt`
- `COLLECTION_IMAGE_ALT`：读取 `Collection.image.altText`
- `ARTICLE_IMAGE_ALT`：读取 `Article.image.altText`

若已非空：

- candidate -> `SKIPPED_ALREADY_FILLED` 或 `RESOLVED`
- 不执行 mutation

**Step B：按平面路由写回**

- `FILE_ALT`：`fileUpdate`
- `COLLECTION_IMAGE_ALT`：`collectionUpdate`
- `ARTICLE_IMAGE_ALT`：`articleUpdate`

**Step C：成功 / 失败收敛**

成功：

- `alt_draft.final_text` 固化
- 写 `audit_log`
- candidate -> `WRITTEN`
- target `current_alt_text`、`current_alt_empty` 立即更新为乐观一致
- decorative mark 若存在则保留，但该项不再计入 decorative

失败：

- candidate -> `WRITEBACK_FAILED_RETRYABLE`
- 记录错误码与错误信息
- 支持重试
