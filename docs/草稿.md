# AGENTS.md

## 项目简介
Shopify Admin Embedded App，旨在为 Shopify 商家自动生成图片的 Alt Text（替代文本）。

## 技术栈 (Stack)
- **框架**: Shopify embedded app, React Router
- **语言**: TypeScript
- **数据库与 ORM**: Prisma + PostgreSQL
- **队列与缓存**: Redis + BullMQ
- **UI 组件库**: Polaris Web Components + App Bridge

## 工作流程与文档规范
1. **基础文档读取**：开始任务前，必须读取 `docs/Specs/tree.txt` 和 `docs/STATUS.md`。
2. **按需查阅文档**：根据当前任务自行判断需要查阅哪些 `docs/Specs` 目录下的 Markdown 文档（**仅限 `docs/Specs` 目录！**）。
3. **状态同步**：任务完成后，将已完成的工作最简明扼要地记录到 `docs/STATUS.md`。
4. **严禁脑补**：不要根据没读过的文档脑补任何信息。
5. **冲突处理**：若文档之间或与当前实现存在冲突，必须明确指出冲突内容。
6. **访问限制**：未经明确指示（如“读取XX文件”），严禁主动读取或检索任何非程序运行必需的文档类文件（如 `.md`, `.txt`, `.pdf` 等）。

## 编码规范与最佳实践

### 1. 语言与类型
- 全面使用**中文**进行注释和文档编写。
- 必须开启 **TypeScript 严格模式**，坚决避免使用 `any` 类型。

### 2. 代码隔离与架构
- 仅服务端代码必须放在 `server/` 目录下，且文件名必须以 `.server.ts` 结尾。
- **严禁**将仅服务端代码（server-only code）移动或导入到客户端可访问的模块中。
- 所有函数遵循**单一职责原则**。
- 每个文件应聚焦于一个明确的职责，或一个紧密相关的领域概念。
- 为新建的非平凡文件（Non-trivial files）添加文件头说明。

### 3. 配置与日志
- 使用 `zod` 进行环境变量的校验。
- 使用 `pino` 实现结构化日志记录。

### 4. Webhook 处理规范
- **Webhook HTTP 处理器（Handler）必须保持极度轻量**，严禁在 handler 中直接处理业务逻辑。
- Handler 仅负责以下四个步骤，完成后必须立即返回 200 HTTP 状态码：
  1. 身份验证
  2. 幂等持久化
  3. 加入队列 (BullMQ)
  4. 立即返回 200
- 实际的 Webhook 业务逻辑必须由独立的 **Worker** 异步处理。
