## 问题原因

错误很明确：**`position` 字段根本不存在于 `MediaImage` 类型上**。

`MediaImage` 实现的是 `Media` 接口，根据 Shopify 官方文档，`Media` 接口及其实现类型（包括 `MediaImage`）的字段只有：`alt`、`id`、`mediaContentType`、`mediaErrors`、`mediaWarnings`、`preview`、`status` 等，**并没有 `position`**。

---

## 解决方案

你的查询中大概有类似这样的代码：

```graphql
# ❌ 错误写法
... on MediaImage {
  id
  position   # ← 这个字段不存在于 MediaImage
  image { url altText }
}
```

### 方案一：直接删除 `position`（如果不需要）

```graphql
... on MediaImage {
  id
  alt
  image { url altText width height }
  mediaContentType
  status
}
```

### 方案二：用 `position` 的替代方式获取顺序

`position` 是 **REST API** 上 `Image` 对象的概念，在 GraphQL 的 `media` 连接中**没有直接等价字段**。如果你需要媒体顺序，可以：

**利用返回顺序**：Shopify 的 `media` 连接默认按展示顺序返回，ndjson 中的顺序即为 position，自己在解析时加序号：

```ts
// 解析 ndjson 时按顺序赋 position
let position = 1;
for (const media of productMediaList) {
  media.position = position++;
}
```

**或者改用 `product.images` 查询**（仅针对图片，REST 风格兼容），`images` 连接下的 `Image` 类型有 `id`、`url`、`altText`，同样按 position 顺序返回。

---

## 总结

| 类型 | 有 `position` 字段？ |
|------|---------------------|
| `MediaImage` (GraphQL) | ❌ 无 |
| `Media` interface (GraphQL) | ❌ 无 |
| REST API `Image` 对象 | ✅ 有 |
| `product.images` 连接 | 隐式顺序，无显式字段 |

**最快的修复方式：从你的 bulk query 的 `... on MediaImage { }` 片段中删除 `position` 即可。**
