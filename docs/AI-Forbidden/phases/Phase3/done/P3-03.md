## P3-03 完成：`GET /api/scan/status` 状态 API

### 变更文件

1. **[`server/modules/scan/scan.types.ts`](server/modules/scan/scan.types.ts)** — 新增 6 个响应类型定义：
   - `ScanStatusAttempt` — 单个 attempt 精简状态（id/status/parsedRows/lastParseError/时间戳）
   - `ScanStatusTask` — 单个 task 状态（resourceType/status/currentAttemptNo + latestAttempt）
   - `ScanStatusJob` — scan_job 总状态（status/publishStatus/scopeFlags/时间戳/error）
   - `ScanProgressSummary` — Redis 进度摘要（completedTasks/totalTasks/status）
   - `ScanStatusResponse` — 完整响应体（scanJob + tasks + progress + lastPublishedAt）

2. **[`server/modules/scan/catalog/scan-job.service.ts`](server/modules/scan/catalog/scan-job.service.ts)** — 新增 `getScanStatus(scanJobId, shopId)` 服务函数：
   - 并行查询 scan_job（含 shop.lastPublishedAt）、scan_task 列表（含 attempts）和 Redis 进度
   - 校验 scan_job 归属当前 shop，不匹配返回 null
   - 自动提取每个 task 最新 attempt（attemptNo 最大）并序列化为 DTO

3. **[`app/routes/api.scan.status.tsx`](app/routes/api.scan.status.tsx)** — 实现 `GET /api/scan/status?scanJobId=...` 路由：
   - authenticate.admin 鉴权 → shopId 查找 → scanJobId 参数校验 → getScanStatus → JSON 响应
   - 400：缺少 scanJobId；404：shop 或 scan_job 不存在

4. **[`docs/STATUS.md`](docs/STATUS.md)** — 新增 P3-03 完成记录
