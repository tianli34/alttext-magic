## P3-07 完成：流式 NDJSON 解析基础设施

### 交付物总览

#### 1. 通用流式 NDJSON Parser — [`ndjson-stream-parser.ts`](server/modules/scan/catalog/parsers/ndjson-stream-parser.ts)
- `streamNdjsonFromUrl<T>(url, options)` — 从 URL fetch → 流式解析 → 批量 flush
- `streamNdjsonFromFile<T>(filePath, options)` — 从本地文件流式解析（fixture 回放）
- `ParentIdCache<T>` — 通用 `__parentId` 父子行缓存映射类
- 核心：逐行解析 + 每 N 行（默认 500）flush，禁止整文件读入内存
- 支持 `handleRow` callback 模式，不同资源类型传入不同 row handler

#### 2. 通用 Batch Flush 组件 — [`staging.service.ts`](server/modules/scan/catalog/staging.service.ts)
- `flushArticleStaging()` / `flushCollectionStaging()` / `flushMediaFileStaging()` — 各资源类型 staging 批量写入
- `flushProductMediaStaging()` — 按 `kind` 标签分发到 `stg_product` + `stg_media_image_product`
- `countStagingRows()` — 查询指定 attempt 的 staging 行数
- 全部使用 `Prisma.createMany({ skipDuplicates: true })` 处理唯一约束冲突

#### 3. 4 类资源 Parser Callback
- [`article.parser.ts`](server/modules/scan/catalog/parsers/article.parser.ts) — Article 扁平行
- [`collection.parser.ts`](server/modules/scan/catalog/parsers/collection.parser.ts) — Collection 扁平行
- [`files.parser.ts`](server/modules/scan/catalog/parsers/files.parser.ts) — Files (MediaImage) 扁平行
- [`product-media.parser.ts`](server/modules/scan/catalog/parsers/product-media.parser.ts) — Product + MediaImage 父子行，含 `__parentId` 缓存 + `positionIndex` 计数
- [`staging.types.ts`](server/modules/scan/catalog/parsers/staging.types.ts) — 共享类型定义 + 校验工具

#### 4. Fixture 回放入口 — [`replay-fixtures.ts`](scripts/replay-fixtures.ts)
- `npx tsx scripts/replay-fixtures.ts` — 回放全部 fixture
- `--batch-size N` — 自定义批次大小
- `--resource <type>` — 仅回放指定资源

#### 5. parse-bulk Worker 集成
- [`parse-bulk.processor.ts`](worker/processors/parse-bulk.processor.ts) — 按 resourceType 分发 → 流式解析 → staging flush → attempt 状态更新
- [`worker/index.ts`](worker/index.ts) — 注册 parse-bulk worker（concurrency: 2）
