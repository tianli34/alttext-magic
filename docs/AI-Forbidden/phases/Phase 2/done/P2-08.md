Bootstrap 聚合服务已完成实现，包含 3 个文件：

### 新增文件

1. **[`bootstrap.types.ts`](server/modules/bootstrap/bootstrap.types.ts)** — 类型定义
   - `PlanSummary` / `QuotaSummary` — 计划与额度占位
   - `LatestScanStatus` — 最近扫描状态
   - `BootstrapData` — 完整聚合返回体

2. **[`bootstrap.service.ts`](server/modules/bootstrap/bootstrap.service.ts)** — 核心服务 `getBootstrapData(shopId)`
   - 并行获取 shop 数据 + notice 状态 + 最近扫描任务（`Promise.all`）
   - 聚合返回：
     - 计划信息占位（`planCode` 从 `shops.currentPlan` 读取）
     - 额度信息占位（Phase 3 前返回默认值）
     - notice 状态（`needsNoticeAck` + `noticeVersion`）
     - scope 三件套：`scanScopeFlags` / `lastPublishedScopeFlags` / `effectiveReadScopeFlags`（计算值，不入库）
     - 最近扫描状态（无则 `null`）
   - **fresh shop 默认态**：`needsNoticeAck = true`，scope 默认四类全开，`effectiveReadScopeFlags` 全 false，`latestScan = null`
   - shop 不存在时防御性返回默认态

3. **[`api.bootstrap.tsx`](app/routes/api.bootstrap.tsx)** — `GET /api/bootstrap` 路由
   - `authenticate.admin` 鉴权 → shopDomain 查 shopId → 调用 `getBootstrapData` → `Response.json`

### 验收要点

- ✅ fresh shop 返回正确默认态（`needsNoticeAck=true`，四类 scope 全开，`effectiveReadScopeFlags` 全 false，`latestScan=null`）
- ✅ `effective_read_scope_flags` 为计算值，不入库
- ✅ 不依赖 Phase 3 数据也能正常返回（额度为占位默认值）
- ✅ TypeScript 编译无新增错误
