已完成 `scan_notice_ack` 服务实现，包含以下文件：

### 新增/修改文件

1. **[`notice.types.ts`](server/modules/notice/notice.types.ts)** — 类型定义
   - `NoticeStatusResult`：版本检查结果（`needsNoticeAck`、`acknowledgedVersion`、`currentVersion`）
   - `AckNoticeInput`：确认入参（`shopId`、`noticeKey`、`version`、`scopeFlagsSnapshot`、`actor?`）

2. **[`scan-notice-ack.service.ts`](server/modules/notice/scan-notice-ack.service.ts)** — 服务实现
   - `ackNotice(input)` — 使用 Prisma upsert 在 `ScanNoticeAck` 表上幂等写入确认记录
   - `getNoticeStatus(shopId, currentVersion)` — 查询 shop 对当前版本的确认状态
   - `checkNeedsAck(acknowledgedVersion, currentVersion)` — **纯函数**，版本比对逻辑：
     - 无记录（`null`/`undefined`）→ `true`（fresh shop 需确认）
     - 旧版本 → `true`（需重新确认）
     - 当前版本已确认 → `false`

3. **[`scan-notice-ack.service.test.ts`](tests/scan-notice-ack.service.test.ts)** — 单测
   - 覆盖：fresh shop（null/undefined）、旧版本、当前版本已确认、边界情况（空字符串、语义不同但字符串不同等）
   - 测试通过 ✅

### 版本检查逻辑对应验收标准

- ✅ 旧版本确认不会被当成已确认 → `checkNeedsAck('1.0', '1.3') === true`
