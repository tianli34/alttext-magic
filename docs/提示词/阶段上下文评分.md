# AltText Magic — 分阶段开发计划 v1.0

---

## 1. 项目概览

**AltText Magic** 是一款面向 Shopify 中小商家的嵌入式 App（Embedded App），通过 AI 在"可控、可审阅"的前提下，批量为店铺四类图片资源（产品媒体 / 文件库 / 集合封面 / 文章封面）补齐缺失的 Alt Text。MVP 需完整跑通 **"扫描 → 生成 → 审阅 → 写回"** 安全闭环，并支持装饰性图片标记、共享文件影响范围提示、Freemium 五档计费与超额包、付费计划专属增量扫描等能力。

**技术栈**：Node.js + TypeScript / React Router（Web）/ Prisma + PostgreSQL / Redis + BullMQ / Shopify App Bridge + Polaris / AI Gateway（主模型 + 降级模型）。部署目标为 Railway（web + worker + postgres + redis）。MVP 核心交付范围覆盖架构文档 §1.1 所列全部 21 项必须做项，以及 §1.2 非功能约束。

# AltText Magic - Phase 1 开发上下文(C)

## 1. 项目定位与技术栈
- **类型**: Shopify Admin Embedded App (面向中小商家)
- **前端**: React Router + Shopify App Bridge + Polaris
- **后端**: Node.js + TypeScript
- **数据库**: PostgreSQL (Prisma ORM)
- **队列**: Redis + BullMQ
- **部署环境**: Railway，需规划 4 个 Service (`web`, `worker`, `postgres`, `redis`)

## 2. Phase 1 核心目标
搭建可部署的 Shopify App 空壳，跑通 **OAuth 安装闭环、Session 持久化、核心基础表初始化、Webhook 安全接收与投递、基础 UI 框架**。暂不实现任何具体的业务（如图片扫描、AI 生成）。

## 3. 数据库与数据模型 (Prisma)
当前阶段**仅需创建 3 张表**：
1. `Session`: 用于 Shopify 官方标准的 Prisma Session Storage。
2. `shops`: 店铺基础信息表。
   - 必含字段：`shop_domain` (主键/唯一键), `installed_at` (安装时间), `uninstalled_at` (卸载时间，可空), `current_plan` (默认值 `FREE`), `scan_scope_flags` (默认配置), `access_token_encrypted` (Offline Access Token 加密存储)。
3. `webhook_event`: 用于 Webhook 幂等处理的记录表。

## 4. 后端核心逻辑要求
- **应用初始化**: 使用 `@shopify/shopify-app-react-router` 模板或类似脚手架初始化。
- **OAuth & Auth**:
  - 实现 Offline Access Token 流程。
  - **安全约束**: Token 落库前必须使用 `AES-256-GCM` 算法加密（密钥通过环境变量注入），读取时解密。
- **Webhook 注册**:
  - 安装时需注册以下 Topic：`APP_UNINSTALLED`, `CUSTOMERS_DATA_REQUEST`, `CUSTOMERS_REDACT`, `SHOP_REDACT` (GDPR三剑客), `BULK_OPERATIONS_FINISH`, `products/create`, `products/update`, `collections/create`, `collections/update`。
- **Webhook 接收与处理管线 (关键)**:
  - 流程：HMAC 校验通过 -> 写入 `webhook_event` 表（实现幂等性） -> **立即返回 200 HTTP 响应** -> 异步投递事件到 BullMQ。
  - `APP_UNINSTALLED` 处理：更新 `shops.uninstalled_at` 字段。
  - GDPR 处理：当前阶段仅占位，直接返回 200 即可。
- **BullMQ 初始化**: 连接 Redis，初始化 Queue 实例。当前阶段 Worker 进程只需启动并处于空循环状态，无需处理实际 Job。
- **日志与环境**: 配置基础的环境变量加载和结构化日志打印。

## 5. 前端核心逻辑要求
- 配置 Shopify App Bridge。
- 使用 Polaris 组件库构建应用外壳（Shell）。
- **页面占位**: 建立左侧/顶部导航结构，包含以下空白占位页面路由：
  - Dashboard
  - Review (审阅)
  - History (历史记录)
  - Billing (计费)
  - Settings (设置)
  - Help (帮助)

## 6. 架构与部署 (Railway) 规划
- **Web 服务**: 运行 React Router 及其底层 Node.js Server（处理 Auth、Webhook 接收、API 和前端渲染）。
- **Worker 服务**: 独立进程，运行 BullMQ 处理器（当前为空跑）。

## 7. AI 验收标准 (Do Definition of Done)
